Python自动化测试

发表于 2018-01-05 | 分类于 Python
字数统计: 543 | 阅读时长 ≈ 2

搞这个Appium自动化测试断断续续也有两周的时候，很早之前也了解过，但是没有实际的应用，这次算是真正的项目实践。

由于本公司秉承开发即测试的原则，所有项目上线后容易出现bug，可能也是由于自己以前太依赖测试，对这方面不怎么注意。还好现在意识到问题的重要性，于是慢慢的自己看是接触点测试的工作，这样更便于注意写代码。

在之前的公司，也用过Xocde自带的单元测试，这个比较适合逻辑代码的验证和网络请求，目前的公司由于比较赶时间，这些就没有施行，但是后续问题慢慢的也就出现了，因为项目比较大且复杂，最主要改动很频繁，需求变化快，这就很尴尬了，每次测试过的地方还要再次测试，还容易出现bug，所以自己就搞了这个自动化测试。

遇到的问题

1.We wanted {“required”:[“value”]} and you sent [“value”,”text”,”id”,”sessionI

上面这个问题是我在测试登录模块的时候遇到的，当弹出键盘，要在输入框输入内容时，脚本停止，然后出现这个错误，Google了下，（顺便一提，能用Google最好，可以省下你很多时间）说是selenium版本的问题，把selenium版本降到3.0.1 就可以了。

首先到python的目录里面找一下selenium的路径（pip show selenium）=> /Library/Python/2.7/site-packages
删除老的版本，然后终端安装的版本pip install selenium==3.0.1
(PS:3.5的版本路径/Library/Frameworks/Python.framework/Versions/3.5/lib/python3.5/site-packages)

我的电脑安装了两个版本 一个是python2.7 这个是Mac自带的
路径是/Library/Python/2.7

还有一个是自己安装的Python3版本
路径是/Library/Frameworks/Python.framework/Versions/3.5/

2.Appium 在 ios 上用 appium 查看页面元素，发现有的元素的 name 属性为 AX error -25205

这个问题是因为升级Xcode9导致的，只能用iOS11的手机去跑
