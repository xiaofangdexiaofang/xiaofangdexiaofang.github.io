iPhoneX和iOS 11的适配

发表于 2017-11-28 | 分类于 iOS开发
字数统计: 334 | 阅读时长 ≈ 1

iPhone X 的导航栏的高度是88，状态栏的高度是44，底部Tabbar的高度是83，圆角以下的距离为34

因为我们这边项目采用的是代码布局，所以适配iPhone X的，修改的代码比较多，主要是以下几个问题：

1.适配导航栏的高度和底部Tabbar的距离，适配安全区域。

2.在Tabbar页面push到其他页面的时候，Tabbar会上移，修改方法是继承UINavigationController创建导航控制器，重写push方法。代码如下：

 -(void)pushViewController:(UIViewController *)viewController animated:(BOOL)animated{


    if (self.viewControllers.count > 0) {
viewController.hidesBottomBarWhenPushed = YES;
}

[super pushViewController:viewController animated:animated];

//修改tabbar的frame
    CGRect frame = self.tabBarController.tabBar.frame;
frame.origin.y = [UIScreen mainScreen].bounds.size.height - frame.size.height;
self.tabBarController.tabBar.frame = frame;

}

3.在wkwebView的 - (void)webView:(WKWebView )webView decidePolicyForNavigationAction:(WKNavigationAction )navigationAction decisionHandler:(void (^)(WKNavigationActionPolicy))decisionHandler 方法中执行到
decisionHandler(WKNavigationActionPolicyAllow) 的方法会崩溃。在iOS 11 之前是这种方法不会崩溃，崩溃的原因是decisionHandler(WKNavigationActionPolicyAllow) 代码调用多次，然后调整代码后，使其只调用一次。

4.在Assets.xcassets中的LaunchImage中添加一个1125x2436像素的启动图。

5.iOS 11 取消了automaticallyAdjustsScrollViewInsets,在项目中加入以下代码

if (@available(ios 11.0,*)) {
    UIScrollView.appearance.contentInsetAdjustmentBehavior = UIScrollViewContentInsetAdjustmentNever;
}else{
self.automaticallyAdjustsScrollViewInsets = false; 
}

6.Xcode9 打包构建新版本的时候要加入1024*1024 AppStore icon

以上就是目前适配遇到的问题。
