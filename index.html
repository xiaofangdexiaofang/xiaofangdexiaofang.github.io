直播模块

发表于 2019-06-22 | 分类于 直播
字数统计: 755 | 阅读时长 ≈ 3

目前在一家新零售的公司做直播模块，包括iOS和安卓。

直播推拉流技术

####主播端

1.音视频采集，使用的是AVFoundation

2.视频处理，GPUImage美颜，水印等

3.音视频编码压缩，包括软编码和硬编码

4.推流，主要走协议（RTMP，HLS，FLV）

####服务器端

1.数据分发

2.截屏

3.录制

4.转码等

####用户端

1.拉流（RTMP，HLS，FLV）

2.解码

3.播放

我在开发的过程中，使用过LFLiveKit，AlivcLivePusher，TXLiteAVSDK_Smart 这三个推流SDK，播放器用的ijkplayer。
说实话技术上没啥难点，都是用别人的东西，看文档的时候认真点就行，这边说几个注意点吧。

注意点

1.阿里的美颜只有基础美颜不收费，腾讯的美颜都收费，没基础版本

2.主播分享当前直播的H5链接的时候，需要跳转到其他APP，需要在切换前台和后台的方法继续和暂停当前的推流，以防止回到直播页面的时候，界面会卡住

3.当切换APP的时候，可以给当前的推流设置一个图片，显示当前主播暂时离开

4.因为是在RN中嵌入原生的模块，会导致内存暴涨，虽然没没弄清为何会这样，但是我这边做处理了，当在直播的时候，收到内存警告是，要清除当前的缓存

5.当你推流的时候，发现使用其中某一个人的账号时每次都闪退，但是用其他人的都没问题，而且报错的原因是推流SDK内部发生错误，[AlivcLivePreview onConnectionLost]，那么基本上就是你这个人的推流地址被禁止了，只需要在阿里云的后台恢复下即可

6.阿里推流的视频都可以被保存下来的，直接存储到OSS对象存储里面的，阿里云的直播管理系统配置下就行

直播聊天

聊天用的是融云的产品，主要就是直播聊天室
也没啥难点，就是看文档的时候认真点就行

注意点

在iOS的SDK中自己给自己发送消息是能接受到的，但是Andriod试了好多会都不行，提了个工单问下技术说可能被拦截了，所以接受不到消息

直播计费

因为阿里官网没有具体的算法，所以自己大概计算了下

假设目前是码率是800kbps,同时在线1000人的条件，直播一个小时

需要的带宽 = 1000*0.8Mbps，大约800M带宽

按日峰值带宽

按照当前这个区间段的价格一天的费用在800*0.638 = 510.4元

按照流量计算

此处注意下，流量和带宽中间差了8倍

流量 = 带宽*时间

流量 = (800Mbps/8)*3600 = 360000MB =352GB

按照这个区间段的价格一天的费用在352*0.264 = 93元
